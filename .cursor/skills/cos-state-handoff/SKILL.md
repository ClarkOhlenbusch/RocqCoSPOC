# CoSProver Agent Skill — Coq Tactic Generation & Proof Formalization

## Your Mission

You have been given a **Chain of States (CoS)** — a sequence of intermediate
proof states (State 0, State 1, ... State N) generated by the Gemini gem from a
formal theorem statement and a Coq-friendly informal proof.

**Your job is to fully formalize that chain into a working Coq proof.**

This means:
1. Taking the CoS (the blueprint) and turning it into real Coq tactic script.
2. Writing tactics that drive the proof from each State N to the next State N+1.
3. Compiling the proof with `coqc` until it accepts the whole file with no errors
   and no `admit.` placeholders remaining.
4. The proof is **not done** until `coqc` exits with code `0` and zero `admit`s.

You are not just suggesting tactics — you are the agent responsible for
delivering a fully compiled, verified proof. Do not stop until `coqc` confirms it.

## 1. Proof-building protocol (MANDATORY)

Build every proof **one tactic step at a time**. Never write more than one
`assert`/tactic block before compiling.

```
loop:
  1. Snapshot the current proof state  (§2)
  2. Choose ONE tactic / assert block  (§3, §4)
  3. Write it into the .v file, followed by `admit.` for everything remaining
  4. Compile with `coqc <file>`
  5. If error  → apply ETR (prompt #4) or fix directly, then goto 1
  6. If success → remove the `admit`, goto 1 for the next step
  7. When no goals remain, remove all `admit.` and do a final `coqc` check
```

**Never skip the compile step.** Blind multi-step writing is the #1 cause of
cascading syntax errors that are hard to diagnose.

## 2. State snapshot (proof-state visibility)

Before choosing any tactic, obtain the current proof state:

```powershell
python .\scripts\get-proof-state.py --file-path <file> --cursor-line <line>
```

- If the output is `No Goals` → the proof is finished or the cursor is outside
  a proof. Stop tactic generation.
- If the output is a `State X:` block → treat it as `state_p`.
- The target state (`state_n`) comes from the Gemini-generated chain of states.

If `coqtop` cannot be found, pass an explicit path:

```powershell
python .\scripts\get-proof-state.py --file-path ... --cursor-line ... --coqtop <path\to\coqtop.exe>
```

## 3. Tactic whitelist for this project

This project uses **Coq 8.20** with `ZArith`, `Znumtheory`, and `Lia`.
Only use tactics known to exist in this environment:

| Safe tactics          | Notes                                              |
|-----------------------|----------------------------------------------------|
| `intros`              | introduce hypotheses                               |
| `assert (H : T)`     | introduce intermediate claim — see §4 for syntax   |
| `apply`               | apply a lemma to the goal                          |
| `exact`               | provide an exact proof term                        |
| `rewrite`, `rewrite <-` | rewrite with an equality                        |
| `replace ... with ... by ring` | algebraic rearrangement (NOT `change`)    |
| `destruct`            | case split / unpack existential                    |
| `exists`              | provide witness for existential                    |
| `lia`                 | linear integer arithmetic                          |
| `nia`                 | nonlinear integer arithmetic                       |
| `ring`                | ring equalities                                    |
| `auto`                | basic automation                                   |
| `split`               | split conjunction goal                             |
| `left` / `right`      | choose disjunct                                    |
| `unfold`              | unfold a definition                                |
| `simpl`               | simplify                                           |

**DO NOT USE** (not available or wrong environment):

| Forbidden              | Why                                                |
|------------------------|----------------------------------------------------|
| `linarith`             | not loaded — use `lia` instead                     |
| `convert ... using N`  | Mathlib (Lean) tactic, not in standard Coq         |
| `ring_nf`              | not available in this Coq                          |
| `have H : T := by`     | Lean syntax — use `assert (H : T). { ... }` instead |
| `change` for algebra   | only works for definitional equality, not ring     |
| `omega`                | deprecated — use `lia`                             |

## 4. Syntax rules — avoiding parser traps

### 4a. Divisibility notation `( _ | _ )`

The notation `(w | x)` adds its own parentheses. Inside `assert`, this means
you need **double closing parens**:

```coq
(* CORRECT *)
assert (Hdiv : (w | (a - b) * E)).

(* WRONG — missing paren, causes "')' expected after [lconstr]" *)
assert (Hdiv : (w | (a - b) * E).
```

**Rule**: After writing any `assert` involving `( _ | _ )`, count parens.
The pattern is always `assert (NAME : (LHS | RHS)).` — note `))` before the `.`.

If in doubt, use the explicit form to bypass the notation entirely:

```coq
assert (Hdiv : Z.divide w ((a - b) * E)).
```

### 4b. Proof blocks

Always use the `assert` + `{ ... }` block style:

```coq
assert (H : PROPOSITION).
{
  tactic1.
  tactic2.
}
```

Do **not** use:
- `have H : T := by ...` (Lean)
- `assert (H : T) by tactic.` for multi-line proofs (fragile with bullets)

### 4c. `replace` vs `change`

- **`replace A with B by ring.`** — for algebraic rearrangements where
  `A` and `B` are propositionally equal (provable by `ring`).
- **`change A.`** — only for definitionally equal terms (unfolding).

If `change` gives `Error: Not convertible`, switch to `replace ... by ring`.

### 4d. Lemma application — `apply` vs `exact`

- `exact lemma_name.` — when the lemma's type matches the goal exactly.
- `apply lemma_name.` — when the lemma's conclusion matches but has premises
  that become subgoals.
- When a lemma takes arguments (e.g., `Z.mod_0_l : forall a, a <> 0 -> 0 mod a = 0`),
  use `apply` and solve subgoals, not `exact`.

### 4e. Left vs Right lemma variants

Before using a rewrite lemma like `Z.mul_0_l` vs `Z.mul_0_r`, check the
goal shape by snapshotting the state first:

- `Z.mul_0_l : forall n, 0 * n = 0`  → use when the goal has `0 * _`
- `Z.mul_0_r : forall n, n * 0 = 0`  → use when the goal has `_ * 0`

When unsure, run `Search` or `Check` in coqtop:

```powershell
echo "Require Import ZArith. Check Z.mul_0_l." | coqtop -quiet
```

## 5. Prompt routing

| Situation                                | Prompt | Inputs                          |
|------------------------------------------|--------|---------------------------------|
| Normal state transition                  | #3     | `state_p`, `state_n`            |
| Coq error on a tactic                    | #4     | `state_p`, `state_n`, error msg, failed tactics |
| Tactics succeed but reach wrong state    | #5     | `state_a`, `state_b` (target), `state_c` (actual) |

## 6. Pre-flight checklist (run before EVERY tactic choice)

1. **Snapshot** the current proof state (§2).
2. **Identify** which chain-of-states transition you're on (State N → State N+1).
3. **Check** that the snapshot matches the expected `state_p` from the chain.
   If it doesn't, use ESR (prompt #5).
4. **Choose** a single tactic from the whitelist (§3).
5. **Write** the tactic + `admit.` placeholder.
6. **Compile** with `coqc`.
7. **Snapshot** the new state to confirm it matches `state_n`.

## 7. Common patterns for this codebase

### Modular arithmetic → divisibility

```coq
assert (H : (x - y) mod w = 0).
{ rewrite Zminus_mod, Hmod, Z.sub_diag. apply Z.mod_0_l. exact Hw. }
assert (Hdiv : (w | (x - y))).
{ apply Zmod_divide. - exact Hw. - exact H. }
```

### Algebraic simplification of divisibility

```coq
assert (Hdiv2 : (w | (a - b) * E)).
{ replace ((a - b) * E) with (x - y) by ring. exact Hdiv1. }
```

### Gauss's lemma (coprimality cancellation)

```coq
assert (Hrel : rel_prime w E).
{ exact (proj1 (Zgcd_1_rel_prime w E) Hgcd). }
rewrite Z.mul_comm in Hdiv2.
apply Gauss with (a := w) (b := E) (c := a - b) in Hdiv2; auto.
```

### Bounded multiple argument

```coq
destruct Hdiv as [t Ht].
assert (Hbound : -w < t * w < w) by (rewrite <- Ht; lia).
destruct (Z.eq_dec t 0) as [Ht0|Ht0].
- rewrite Ht0, Z.mul_0_l in Ht. lia.
- assert (Hw_pos : 0 < w) by lia. nia.
```
